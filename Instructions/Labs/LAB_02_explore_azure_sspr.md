# 랩: Microsoft Entra 셀프 서비스 암호 재설정

이 랩에 해당하는 Learn 학습 내용은 다음과 같습니다.

- 학습 경로: Microsoft Entra의 기능 설명
- 모듈: Microsoft Entra ID의 인증 기능 설명
- 단원: 셀프 서비스 암호 재설정 설명

## 랩 시나리오

이 랩에서는 관리자로 로그인하여 Microsoft 365 테넌트에 이미 설정되어 있는 SSPR 보안 그룹에 사용자를 추가하는 과정을 진행합니다. SSPR을 사용하도록 설정한 후에는 사용자 역할로 로그인하여 SSPR에 등록하고 암호를 재설정하는 프로세스를 진행합니다.  마지막으로 다시 관리자로 로그인하여 SSPR의 감사 로그 및 사용 데이터와 인사이트를 확인합니다.

**예상 시간:** 30분

### 작업 1

이 작업에서는 관리자로 로그인하여 SSPR에 사용 가능한 몇 가지 구성 설정을 살펴봅니다.

1. Microsoft Edge 브라우저를 엽니다. 주소 표시줄에 **https://entra.microsoft.com**을 입력한 후 ALH(권한 있는 랩 호스터)에서 제공한 Microsoft 365 관리자 자격 증명을 사용하여 로그인합니다.
    1. 로그인 창에 **admin@WWLxZZZZZZ.onmicrosoft.com**을 입력하고(여기서 ZZZZZZ는 ALH에서 제공한 고유 테넌트 ID) **다음**을 선택합니다.
    1. 랩 호스팅 공급자가 제공한 관리자 암호를 입력합니다. **로그인**을 선택합니다.
    1. 랩 호스터에 따라 테넌트에 처음 로그인하는 경우 MFA 등록 프로세스를 완료하라는 메시지가 표시될 수 있습니다. , 화면의 프롬프트에 따라 MFA를 설정합니다.
    1. 로그인하면 Microsoft 365 관리 센터 페이지로 이동됩니다.

1. 왼쪽 탐색 창에서 **보호** 옵션을 확장하고 **암호 재설정**을 선택합니다.  

1. 셀프 서비스 암호 재설정의 속성이 표시됩니다. **셀프 서비스 암호 재설정** 옆의 정보 아이콘을 선택하여 설명 내용을 확인합니다. **선택됨**이 파란색으로 강조 표시되어 있는지 확인합니다. 이제 **그룹 선택** 옆의 정보 아이콘 위에 커서를 놓으면 "암호를 재설정할 수 있는 사용자 그룹을 정의합니다."라는 설명이 표시됩니다. 해당 그룹에 사용자를 포함해야 하며 사용자를 개별적으로 선택할 수는 없습니다. Microsoft 365 테넌트 내에 미리 구성되어 있는 SSPRSecurityGroupUsers 그룹이 이미 표시되어 있습니다. 마지막으로, "이 설정은 조직 내 최종 사용자에게 적용됩니다."라는 메시지가 표시된 연한 파란색 상자를 확인합니다. 관리자는 항상 셀프 서비스 암호 재설정을 진행할 수 있으며, 두 가지 인증 방법을 사용하여 암호를 재설정해야 합니다.

1. 암호 재설정의 왼쪽 탐색 패널에서 **인증 방법**을 선택합니다.

1. 재설정에 필요한 방법 수에서 **1**을 선택합니다. 화면에 표시되는 정보 상자에서

1. 사용자에게 제공되는 다양한 방법을 살펴봅니다.  **이메일**과 **휴대폰**이 이미 선택되어 있어야 합니다. 이 두 옵션이 선택되어 있지 않으면 선택합니다.

1. 암호 재설정의 왼쪽 탐색 패널에서 **등록**을 선택합니다.  

1. 로그인 시 사용자가 등록을 해야 합니까? 설정이 **예**로 설정되어 있는지 확인합니다.  사용자에게 인증 정보를 다시 확인하도록 요청하기까지의 기간은 기본값인 **180**으로 유지합니다.   페이지에 표시되는 정보 상자를 살펴봅니다.

1. 암호 재설정의 왼쪽 탐색 패널에서 **알림**을 선택합니다.  

1. 암호가 재설정되는 경우 사용자에게 알리시겠습니까? 설정이 **예**로 설정되어 있는지 확인합니다.  다른 관리자가 암호를 재설정하는 경우 모든 관리자에게 알리시겠습니까? 설정은 **아니요**로 유지합니다.

1. 암호 재설정 탐색 창에는 감사 로그 보기 옵션과 사용량 및 인사이트 옵션도 포함되어 있습니다.

1. 창 오른쪽 위에서 **X**를 선택하여 암호 재설정 창을 닫습니다. 그러면 Microsoft Entra 관리 센터가 다시 표시됩니다.

1. Microsoft Entra 창을 열어 두세요.

### 작업 2

이 작업에서는 관리자로 로그인하여 이전 랩 연습에서 만든 사용자를 SSPR 보안 그룹에 추가합니다.

1. 브라우저 탭을 열고 Microsoft Entra 관리 센터 **[entra.microsoft.com](https://entra.microsoft.com)** 홈 페이지로 이동합니다. 필요한 경우 **ID**를 확장합니다.

1. 왼쪽 탐색 패널의 "ID" 아래에서 **그룹**을 확장하고 **모든 그룹**을 선택합니다.

1. 기존 그룹 목록이 표시됩니다. 그룹 검색 필드에 **SSPR**을 입력하고 검색 결과에서 **SSPRSecurityGroupUsers**를 선택합니다.  그러면 이 그룹의 구성 옵션이 표시됩니다.

1. 왼쪽 탐색 창에서 **구성원**을 선택합니다.

1. 페이지 위쪽에서 **+ 구성원 추가**를 선택합니다.  

1. 검색 상자에 **Sara Perez**를 입력합니다.  **Sara Perez** 사용자가 검색 상자 아래에 표시되면 해당 사용자를 선택하고 페이지 아래쪽의 **선택**을 누릅니다.  그러면 구성원 페이지가 다시 표시됩니다.  페이지 위쪽에서 **새로 고침**을 선택합니다. 이제 Sara Perez가 SSPR 보안 그룹 구성원으로 표시됩니다.

1. 화면 오른쪽 위의 이메일 주소 옆에 있는 사용자 아이콘을 클릭하여 모든 브라우저 탭에서 로그아웃합니다. 그런 다음 브라우저 창을 모두 닫습니다.

### 작업 3

이 작업에서는 Sara Perez 사용자로 로그인하여 셀프 서비스 암호 재설정 등록 프로세스를 진행합니다.

1. Microsoft Edge를 열고 주소 표시줄에 **https://login.microsoft.com**을 입력합니다.

1. Sara Perez로 로그인합니다. 로그인 프로세스에는 MFA가 필요할 수 있습니다.

1. 추가 정보가 필요하다는 팝업이 표시됩니다.  Adele Vance는 SSPRSecurityGroupUsers 그룹의 구성원인데, 그룹의 구성에 따라 해당 구성원은 로그인할 때 등록을 해야 하기 때문입니다.  **다음** 버튼을 선택합니다.  참고: 관리자가 사용자를 추가할 때 인증 방법을 직접 구성하여 사용자가 직접 등록을 진행하도록 할 수도 있습니다. 이렇게 하려면 관리자는 사용자가 셀프 서비스 암호 재설정을 수행하여 사용자 암호를 재설정하는 데 사용하는 전화 번호와 이메일 주소를 알고 이러한 정보를 설정해야 합니다.

1. "계정 보안 유지" 페이지가 열립니다.  표시되는 창과 그 후의 단계는 Microsoft Authenticator 앱 메서드용입니다. 이메일을 대신 사용하려면 **다른 방법을 설정하고 싶습니다**를 선택하고 단계를 따르세요.
    1. 모바일 디바이스에 이미 Microsoft Authenticator 앱을 설치한 경우 **다음**을 선택합니다. 그렇지 않은 경우 **지금 다운로드**를 선택하고 단계를 따릅니다.
    1. 계정 설정을 시작하게 됩니다.  **다음**을 선택합니다.
    1. 모바일 디바이스의 Microsoft Authenticator 앱을 사용하여 **+** 선택을 통해 계정을 만들고 **회사 또는 학교 계정**을 선택합니다.
    1. **QR 코드 스캔** 옵션을 선택하고 모바일 디바이스를 사용하여 PC 화면의 QR 코드를 스캔합니다.
    1. PC 및 모바일 디바이스에서 단계를 수행하고 **다음**을 선택합니다.
    1. 보안 정보를 설정하면 성공 창이 표시됩니다.  **완료**를 선택합니다.

1. 이제 로그인을 완료할 수 있습니다. 로그인 시간이 만료된 경우 암호를 다시 입력하세요.

1. 화면 오른쪽 위의 이메일 주소 옆에 있는 사용자 아이콘을 클릭하여 모든 브라우저 탭에서 로그아웃합니다. 그런 다음 브라우저 창을 모두 닫습니다.

### 작업 4(선택 사항)

이 작업에서는 Sara Perez 사용자로 로그인하여 암호 재설정 프로세스를 진행합니다.

1. Microsoft Edge를 엽니다.

1. 주소 표시줄에 **https://login.microsoft.com**을 입력합니다.

1. 이메일 주소 **sara@WWLxZZZZ.onmicrosoft.com**을 입력하여(여기서 ZZZZZZ는 랩 호스팅 공급자가 제공한 고유 테넌트 ID) Sara Perez로 로그인하고 **다음** 단추를 선택합니다. 계정 선택 창이 열릴 수도 있습니다. 그러면 Sara Perez의 계정을 선택합니다.

1. 암호 입력 창에서 **암호를 잊어버렸음**을 선택합니다.

1. 계정으로 돌아가기 창이 열립니다.   Sara Perez의 이메일 주소인 sara@WWLxZZZZ.onmicrosoft.com이 이메일 또는 사용자 이름 상자에 표시되는지 확인합니다.  해당 이메일이 표시되지 않으면 상자에 이메일을 입력합니다.  

1. 이미지에 표시되어 있는 문자나 오디오에서 들리는 단어를 비어 있는 상자에 입력합니다. 해당 정보를 입력한 후 **다음**을 선택합니다.

1. 화면에 계정으로 돌아가기가 표시되고 확인 1단계 > 새 암호 선택이 표시됩니다. **Authenticator 앱 알림 승인** 옵션을 선택한 다음 **알림 보내기**를 선택합니다.

1. PC의 숫자를 확인하고 지침에 따라 모바일 디바이스에서 Microsoft Authenticator 앱을 사용하여 로그인을 승인합니다.

1. 다음 화면에 새 암호를 입력하고 확인하라는 메시지가 표시됩니다.  새 암호와 확인 암호를 입력하고 **마침** 단추를 선택합니다.

1. 암호가 재설정되었다는 메시지가 화면에 표시됩니다.  **여기를 클릭하세요.** 를 선택하여 새 암호로 로그인합니다.

1. 계정 정보 선택 상자에 **sara@WWLxZZZZZZ.onmicrosoft.com**과 새 암호를 입력하고 **로그인** 단추를 선택합니다.  로그인 상태를 유지할지 묻는 메시지가 표시되면 **아니요**를 선택합니다.

1. 이제 Sara의 Microsoft 계정에 로그인되었습니다.

1. 화면 오른쪽 위의 이메일 주소 옆에 있는 사용자 아이콘을 선택한 후 **로그아웃**을 선택하여 로그아웃합니다. 그런 다음 브라우저 창을 모두 닫습니다.

### 작업 5(선택 사항)

이 작업에서는 관리자로 로그인하여 암호 재설정과 관련된 감사 로그와 사용량 및 인사이트 데이터를 간략하게 살펴봅니다.

1. Microsoft Edge를 엽니다.

1. 주소 표시줄에 **https://entra.microsoft.com**을 입력한 후 ALH(권한 있는 랩 호스터)에서 제공한 Microsoft 365 관리자 자격 증명을 사용하여 로그인합니다.

1. Microsoft Entra 관리 센터가 표시됩니다.  왼쪽 탐색 창에서 **보호** 옵션을 확장하고 **암호 재설정**을 선택합니다.

1. 왼쪽 탐색 창에서 **감사 로그**를 선택합니다.  제공되는 정보와 사용 가능한 필터를 확인합니다.  로그를 다운로드할 수도 있음을 확인합니다.  

1. **다운로드**를 선택합니다.  다운로드 형식은 CSV 또는 JSON으로 지정할 수 있습니다.  화면 오른쪽 위에서 **X**를 선택하여 창을 닫습니다.

1. 왼쪽 탐색 창에서 **사용량 및 인사이트**를 선택합니다.

1. 등록과 관련하여 제공되는 정보를 확인합니다.  데이터를 새로 고친 후에도 이 데이터가 갱신되려면 시간이 다소 걸릴 수 있습니다. 그러므로 이전 작업의 등록 또는 사용량 데이터가 아직 반영되어 있지 않을 수도 있습니다.

1. 페이지 위쪽에서 **사용량**을 선택하여 셀프 서비스 암호 재설정 횟수 및 방법별 계정 잠금 해제 횟수를 확인합니다.  데이터를 새로 고친 후에도 이 데이터가 갱신되려면 시간이 다소 걸릴 수 있습니다. 그러므로 이전 작업의 사용량 데이터가 아직 반영되어 있지 않을 수도 있습니다.

1. 열려 있는 브라우저 탭을 닫습니다.

### 검토

이 랩에서는 관리자로 로그인하여 셀프 서비스 암호 재설정을 사용하도록 설정하는 프로세스를 진행했습니다. SSPR을 사용하도록 설정하면 사용자의 역할로 간주하여 SSPR에 등록하고 암호를 재설정하는 프로세스를 진행하게 됩니다.  마지막으로 다시 관리자로 로그인하여 SSPR의 감사 로그와 사용량 및 인사이트 데이터를 확인했습니다.
